РУКОВОДСТВО ДЛЯ ПРЕПОДАВАТЕЛЯ A-school

Этот файл описывает рабочие процессы преподавателя по текущему коду проекта:
создание домашнего задания, заполнение полугодового плана и отметка посещаемости.
Все URL указаны как относительные пути внутри сайта.


1. ВХОД И ОСНОВНАЯ НАВИГАЦИЯ

1. Войдите в систему как пользователь с ролью TEACHER:
   /login

2. После входа откройте панель управления:
   /dashboard

3. Основная рабочая страница преподавателя - список учеников:
   /teacher/class/

   В боковом меню она называется "Ученики".

4. На странице "Мои занятия" можно:
   - отфильтровать учеников по циклу обучения;
   - посмотреть курс, направление, класс, средний балл и посещаемость;
   - открыть карточку конкретного ученика кнопкой "Открыть карточку".

5. Карточка ученика открывается по URL:
   /teacher/students/<student_id>/

   Здесь находятся основные кнопки:
   - "Добавить задание";
   - "Настроить расписание";
   - "Ввести результат";
   - "Полугодовой план".

Важно: преподаватель видит и редактирует только тех учеников, которые привязаны
к его курсам. Если ученик не привязан к курсу преподавателя, система вернет
ошибку доступа или сообщение о том, что ученик не назначен на ваш курс.


2. СОЗДАНИЕ ДОМАШНЕГО ЗАДАНИЯ

Основной URL для создания задания конкретному ученику:
/teacher/students/<student_id>/assignments/create/

Обычный URL /assignments/create/ для преподавателя перенаправляет в раздел
"Ученики", поэтому рабочий путь начинается с карточки ученика.

Пошаговый процесс:

1. Откройте:
   /teacher/class/

2. Найдите ученика и нажмите "Открыть карточку".

3. В карточке ученика нажмите "Добавить задание".

   Кнопка показывается, когда для ученика однозначно определяется один курс
   преподавателя. Если у ученика несколько ваших курсов, система может попросить
   уточнить курс у администратора.

4. На странице "Новое задание" проверьте строку:
   Ученик: <имя ученика> · Курс: <название курса>

5. Заполните поля формы:

   "Название (опционально)"
   - Можно оставить пустым, если ниже заполнены композиции.
   - Если композиции не указаны, название используется как название задания.
   - В форме действует ограничение: значение должно быть короче 50 символов.

   "Описание"
   - Дополнительный комментарий к заданию.
   - Также должно быть короче 50 символов по текущей валидации формы.

   "Композиции"
   - Основной способ создавать задания по произведениям.
   - Каждая заполненная композиция создает отдельное домашнее задание.
   - Кнопка "+" добавляет еще одну строку композиции.
   - Название композиции должно быть не длиннее 200 символов.

   Если у ученика есть активные, еще не выполненные композиции из предыдущего
   домашнего задания, они автоматически подставляются в список. Их можно
   оставить, изменить или очистить перед сохранением.

6. При необходимости импортируйте пункты из полугодового плана:

   В блоке "Полугодовой план" выберите:
   - "I полугодие";
   - или "II полугодие".

   После выбора полугодия ниже появятся пункты плана, если они уже заведены.
   Отметьте чекбоксы у нужных пунктов. Выбранные пункты будут добавлены к
   списку создаваемых заданий вместе с композициями.

   Прямые URL для выбора полугодия при создании задания:
   /teacher/students/<student_id>/assignments/create/?half_year=H1
   /teacher/students/<student_id>/assignments/create/?half_year=H2

7. Укажите "Дедлайн".

   Это обязательное поле с выбором даты.

8. При необходимости добавьте файл в поле:
   "Прикрепить фото / видео"

9. Нажмите "Создать".

10. Что происходит после сохранения:
   - создается Assignment;
   - создается назначение AssignmentTarget для выбранного ученика со статусом TODO;
   - создается связанная контрольная точка в журнале оценок типа HOMEWORK;
   - для ученика создается пустая оценка, которую преподаватель может заполнить позже.

11. После успешного создания система возвращает в карточку ученика:
   /teacher/students/<student_id>/


2.1. Просмотр и удаление домашних заданий

Общий список домашних заданий преподавателя:
/assignments/

На этой странице преподаватель видит:
- дедлайн;
- курс;
- название задания;
- количество назначений ученикам;
- материалы, если был прикреплен файл.

Дополнительные действия:

1. Фильтр "Цикл учеников" позволяет показать задания только по выбранному циклу.

2. Кнопка "Выбрать ученика" возвращает в:
   /teacher/class/

3. Кнопка "Режим удаления" включает чекбоксы.
   После выбора заданий нажмите "Удалить выбранное".

URL удаления выбранных заданий:
/assignments/bulk-delete/

Удалять можно только задания, созданные текущим преподавателем, либо задания
курса, где текущий пользователь является преподавателем.


2.2. Как ученик завершает задание

Ученик видит свои задания на:
/assignments/

Он может отправить:
- комментарий;
- видео ответа;
- или оба варианта.

После отправки статус назначения становится DONE. Оценка при этом не выставляется
автоматически: преподаватель заполняет результат отдельно.

Результаты по ученику:
/teacher/students/<student_id>/results/

Журнал оценок по курсу:
/teacher/courses/<course_id>/grades/


3. ПОЛУГОДОВОЙ ПЛАН

Полугодовой план хранится как список пунктов Goal для конкретного ученика.

Основной URL списка:
/goals/?student=<student_id>&half_year=H1
/goals/?student=<student_id>&half_year=H2

H1 - I полугодие, месяцы с января по июнь.
H2 - II полугодие, месяцы с июля по декабрь.

Если параметр half_year не передан, система выбирает текущее полугодие по
текущему месяцу.


3.1. Как открыть полугодовой план ученика

1. Откройте:
   /teacher/class/

2. Нажмите "Открыть карточку" у нужного ученика.

3. В карточке ученика нажмите "Полугодовой план".

   Ссылка ведет на:
   /goals/?student=<student_id>&half_year=<H1_or_H2>

4. На странице плана можно переключаться между:
   - "I полугодие";
   - "II полугодие".


3.2. Как добавить пункты полугодового плана

Основной URL создания для преподавателя:
/teacher/students/<student_id>/goals/create/?half_year=H1
/teacher/students/<student_id>/goals/create/?half_year=H2

Пошаговый процесс:

1. Откройте полугодовой план ученика.

2. Выберите нужное полугодие кнопкой "I полугодие" или "II полугодие".

3. Нажмите "Добавить в полугодовой план".

4. На странице создания выберите полугодие.

   Если вы пришли из списка плана, нужное полугодие уже передается в URL, но
   его можно изменить кнопками формы.

5. Заполните поле "Пункт плана".

   Пример:
   - Гамма До мажор двумя руками
   - Этюд N5
   - Работа над динамикой в пьесе

6. Чтобы добавить несколько пунктов за один раз, нажмите "+".

7. Нажмите "Создать".

8. После сохранения система возвращает в список плана:
   /goals/?student=<student_id>&half_year=<H1_or_H2>

Ограничения:
- пустые строки не сохраняются;
- каждый пункт должен быть короче 50 символов по текущей валидации формы;
- преподаватель может добавлять план только ученикам своих курсов.


3.3. Статусы, редактирование и удаление пунктов плана

На странице списка плана каждый пункт имеет статус:
- "В процессе";
- "Выполнено".

Как изменить статус:

1. Откройте:
   /goals/?student=<student_id>&half_year=<H1_or_H2>

2. В строке пункта выберите нужный статус в выпадающем списке.

3. Нажмите "Сохранить".

Технический URL обновления статуса:
/goals/<goal_id>/status/

Как изменить текст пункта:

1. В строке пункта нажмите "Изменить".

2. На странице редактирования можно:
   - поменять полугодие;
   - изменить текст пункта.

URL редактирования:
/goals/<goal_id>/edit/

Как удалить пункты:

1. На странице плана нажмите "Режим удаления".

2. Отметьте чекбоксы у пунктов.

3. Нажмите "Удалить выбранное".

URL массового удаления:
/goals/bulk-delete/


3.4. Связь полугодового плана с домашними заданиями

Пункты полугодового плана используются на странице создания домашнего задания.

Если в плане ученика уже есть пункты, то при создании задания можно:

1. Открыть:
   /teacher/students/<student_id>/assignments/create/?half_year=H1
   или
   /teacher/students/<student_id>/assignments/create/?half_year=H2

2. В блоке "Полугодовой план" отметить нужные пункты.

3. После сохранения каждый выбранный пункт будет добавлен как отдельное
   домашнее задание вместе с другими композициями.


4. ПОСЕЩАЕМОСТЬ И РАСПИСАНИЕ

В текущей версии основной процесс посещаемости построен вокруг регулярного
расписания ученика и сгенерированных уроков-слотов LessonSlot.

Ключевые URL:

Расписание ученика:
/teacher/students/<student_id>/schedule/

Календарь преподавателя:
/calendar/

Отчет по конкретному уроку:
/slots/<slot_id>/report/

Перенос конкретного урока:
/slots/<slot_id>/reschedule/

Журнал посещаемости:
/attendance/


4.1. Как настроить регулярное расписание ученика

1. Откройте:
   /teacher/class/

2. Откройте карточку нужного ученика.

3. Нажмите "Настроить расписание".

   Откроется:
   /teacher/students/<student_id>/schedule/

4. В блоке "Создать регулярный урок" заполните:

   "День недели"
   - выберите день из списка.

   "Номер урока (опционально)"
   - можно оставить пустым;
   - допустимые значения: от 1 до 12.

   "Время начала"
   - обязательное поле формата HH:MM.

5. Чтобы добавить несколько регулярных уроков сразу, нажмите "+" и заполните
   дополнительную строку.

6. Нажмите "Создать урок".

7. После сохранения система:
   - создает или обновляет регулярное расписание StudentSchedule;
   - делает расписание активным;
   - создает будущие уроки-слоты на 60 дней вперед;
   - показывает сообщение с количеством сохраненных регулярных уроков и
     созданных будущих уроков.

Ограничения:
- нельзя отправить полностью пустую форму;
- нельзя добавить одинаковую пару "день недели + время начала" дважды;
- длительность урока в этом потоке фиксирована: 40 минут;
- если у ученика несколько курсов преподавателя, страница показывает уведомление,
  к какому курсу будут привязаны уроки.


4.2. Блок "Регулярные уроки"

После создания расписания на странице:
/teacher/students/<student_id>/schedule/

появится таблица "Регулярные уроки".

В ней отображаются:
- день;
- время;
- номер урока;
- длительность;
- статус "Активно" или "Неактивно".

Кнопка "Отключить уроки":
- делает регулярный урок неактивным;
- удаляет будущие запланированные слоты, которые еще не проведены.

Кнопка "Включить уроки":
- снова активирует расписание;
- заново создает будущие слоты.


4.3. Как заполнить посещаемость из расписания ученика

1. Откройте:
   /teacher/students/<student_id>/schedule/

2. Найдите блок "Ближайшие уроки".

3. В строке нужного урока посмотрите:
   - дату;
   - время;
   - статус;
   - текущую посещаемость.

4. Если урок запланирован на будущую дату, вместо кнопки будет текст:
   "Доступно в день урока".

   Заполнить отчет до дня урока нельзя.

5. В день урока или после него нажмите:
   "Заполнить отчёт".

6. Откроется окно "Посещаемость" с двумя кнопками:
   - "Присутствовал";
   - "Не присутствовал".

7. Нажмите нужную кнопку.

8. После сохранения:

   Если выбрано "Присутствовал":
   - статус урока становится "Проведен" (DONE);
   - посещаемость становится PRESENT.

   Если выбрано "Не присутствовал":
   - статус урока становится "Пропущен" (MISSED);
   - посещаемость становится ABSENT.

9. После сохранения система возвращает обратно на страницу расписания.

Если урок уже заполнен, кнопка называется "Открыть отчёт". Через нее можно
открыть тот же отчет и изменить отметку.


4.4. Как заполнить посещаемость из календаря

1. Откройте:
   /calendar/

2. На странице "Расписание" показана неделя.

3. Используйте кнопки:
   - "Предыдущая неделя";
   - "Текущая неделя";
   - "Следующая неделя".

4. Уроки преподавателя отображаются по дням. У каждого урока видны:
   - время;
   - имя ученика;
   - курс;
   - статус урока;
   - посещаемость, если она уже заполнена.

5. Для доступного урока нажмите:
   "Заполнить отчёт"
   или
   "Открыть отчёт".

6. В окне "Посещаемость" выберите:
   - "Присутствовал";
   - или "Не присутствовал".

7. Сохранение идет на URL:
   /slots/<slot_id>/report/

8. После сохранения система возвращает на ту же неделю календаря.

Важно: при открытии календаря для преподавателя система дополнительно проверяет
активные регулярные расписания и генерирует недостающие слоты на ближайшие дни.


4.5. Прямое открытие отчета по уроку

Если известен slot_id, отчет можно открыть напрямую:
/slots/<slot_id>/report/

На странице будут показаны:
- ученик;
- курс;
- дата и время урока;
- кнопки "Присутствовал" и "Не присутствовал".

Если урок еще в будущем и имеет статус "Запланирован", система не даст заполнить
отчет и покажет сообщение:
"Отчёт можно заполнить в день занятия."


4.6. Как перенести урок

Перенос доступен только для урока со статусом "Запланирован".

Из расписания ученика или календаря нажмите "Перенести урок".

URL:
/slots/<slot_id>/reschedule/

На странице переноса заполните:

1. "Новая дата"
2. "Новое время начала"
3. "Причина переноса (опционально)"

Нажмите "Сохранить перенос".

Система проверяет:
- дата или время действительно изменились;
- у этого ученика нет другого урока на выбранные дату и время;
- урок все еще находится в статусе "Запланирован".

После успешного переноса система возвращает обратно в расписание или календарь.


4.7. Журнал посещаемости

Журнал посещаемости открывается по URL:
/attendance/

Можно также передать курс и месяц:
/attendance/?course=<course_id>&month=YYYY-MM

Пример:
/attendance/?course=3&month=2026-04

Что видно в журнале:
- список дат уроков за выбранный месяц;
- ученики выбранного курса;
- отметки посещаемости по каждому ученику;
- итоги проведенных уроков за месяц.

Обозначения:
- галочка - PRESENT, ученик присутствовал;
- "Н" - ABSENT, ученик отсутствовал;
- "Б" - SICK, ученик болел;
- "О" - LATE, ученик опоздал;
- "-" - урок еще запланирован или отметка не заполнена.

Важно: текущая форма отчета по слоту дает преподавателю две основные кнопки:
"Присутствовал" и "Не присутствовал". Обозначения "Б" и "О" поддерживаются
моделью и журналом, но отдельные кнопки для них в текущем интерфейсе не выведены.

Если у преподавателя один курс, он выбирается автоматически. Если курсов
несколько, на странице появится выбор курса.

Журнал служит для просмотра сводной посещаемости. Для заполнения конкретной
отметки используйте:
- /calendar/
- или /teacher/students/<student_id>/schedule/


4.8. Дополнительная страница "Уроки"

В проекте также есть страница:
/lessons/

Она показывает список уроков, дату, композиции и комментарий.

Для преподавателя на этой странице:
1. Нужно выбрать ученика в фильтре "Ученик".
2. После выбора ученика появляется колонка "Посещение".
3. В ней можно поставить или снять чекбокс "отметка".
4. Затем нажать "Сохранить".

Этот поток работает с моделью LessonStudent и старыми уроками Lesson.
Новый основной поток посещаемости в интерфейсе идет через регулярное расписание,
слоты и отчет:
/teacher/students/<student_id>/schedule/
/slots/<slot_id>/report/


5. КРАТКИЙ СПРАВОЧНИК URL

Вход:
/login

Панель управления:
/dashboard

Список учеников преподавателя:
/teacher/class/

Карточка ученика:

6. ЧАСТЫЕ СИТУАЦИИ

Ситуация: не видна кнопка "Добавить задание".
Проверьте, что ученик привязан к одному курсу преподавателя. Если у ученика
несколько курсов преподавателя, нужно уточнить привязку курса у администратора.

Ситуация: при создании задания ошибка "Укажите название задания или добавьте
хотя бы одну композицию".
Заполните поле "Название" или хотя бы одну строку "Название композиции".

Ситуация: в домашнем задании нет пунктов из плана.
Сначала заполните полугодовой план ученика на /goals/, затем вернитесь на
страницу создания задания и выберите нужное H1 или H2.

Ситуация: кнопка "Заполнить отчёт" не показывается у будущего урока.
Это ожидаемое поведение. Отчет по запланированному уроку доступен только в день
урока или после него.

Ситуация: журнал посещаемости показывает прочерк.
Это значит, что урок еще запланирован или посещаемость по нему не заполнена.

Ситуация: нужно перенести урок.
Используйте "Перенести урок" в календаре или в блоке "Ближайшие уроки" на
странице расписания ученика. Перенос доступен только для запланированных уроков.
